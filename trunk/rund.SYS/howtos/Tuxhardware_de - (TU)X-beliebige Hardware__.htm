WLAN Allnet ALL0281 54 Mbit/s WLAN PCI-Karte 36,00EUR

Technische Daten:

Schnittstelle: PCI
Chipsatz: Atheros 5212
Protokoll: 802.11b und 802.11g
Übertragungsrate: maximal 54 Mbit/s (108 Mbit/s mit Kanalbündelung)
Sicherheit: 64/128 Bit WEP Datenverschlüsselung, WPA-PSK Verschlüsselung
Abmessungen/Gewicht: 121 mm x 134 mm / 85 g

54 Mbit/s PCI-Karte von Allnet

Die 32-Bit 54 Mbit/s Wireless PCI Netzwerkkarte - ALL0271 entspricht dem IEEE 802.11g Draft-Standard und erlaubt Datenraten von bis zu 54 Mbit/s im lizenzfreien 2.4 GHz ISM Frequenzband. Der PCI-Adapter ist dabei 100% abwärtskompatibel zu herkömmlichen 11 Mbit/s Komponenten nach IEEE 802.11b. Die höheren Datenraten werden durch die OFDM Technologie (Orthogonal Frequency Division Multiplexing) erreicht. Für Datensicherheit sorgt 64-/ 128-Bit WEP Verschlüsselung und WPA-PSK Verschlüsselung.

Diese Karte haben wir übrigens auch als PCMCIA-Formfaktor im Angebot.

Bereit für Linux:

Der passende Treiber kommt nicht von ALLNET, sondern aus der OpenSource-Gemeinde. Achtung! Es wurden uns Probleme unter SuSe 9.1 mit dieser Karte berichtet. Wir klären derzeit, wo das Problem liegt. In unseren Tests gab es keine Beanstandungen mit der Kernel-Version 2.6 auf der auch Suse 9.1 basiert.

Alternativ bietet die Firma Linuxant kostenpflichtige Treiber an (Download ist kostenlos, license key muss online gekauft werden) die zB. auch mit dem Kernel der SuSE 9.0 ohne weiteres funktionieren. Ganz allgemein gesagt ist es mit diesen Treiber möglich fast jede im Moment auf dem Markt erhältliche Karte unter Linux zu betreiben. Linuxant berechnet beim jetzigen Dollar-Kurs ca. 15EUR für diesen Treiber.

Installation unter Linux:

siehe auch das README-File des madwifi-Treiber.

  • Kernel-Support für WLAN muss aktiviert sein (hier am Beispiel eines 2.6er Kernels): 
    Device Drivers  --->
    Networking support  --->
        Wireless LAN (non-hamradio)  --->
            [*] Wireless LAN drivers (non-hamradio) & Wireless Extensions
  • Nachdem der Kernel neu kompiliert worden ist, muss der heruntergeladene Treiber kompiliert werden.
  • Laden der module (in dieser Reihenfolge): 
    modprobe wlan
modprobe ath_hal
modprobe ath_pci
  • Das Kommando iwconfig lässt erkennen, unter welchem Devicenamen die Karte eingebunden wurde.

Betrieb unter Linux:

Folgende Kommados sind hilfreich:

  • um die von der Karte erkannten Access-Points aufzulisten:
    iwlist ath0 scan
  • um die SSID auf "foo" zu setzen:
    iwconfig ath0 essid "foo"
  • um den Betrieb der Karte auf 11b zu setzen:
    iwpriv ath0 mode 2
  • um den Betrieb der Karte auf 11g zu setzen:
    iwpriv ath0 mode 3

Folgendes Skript erzeugt einen zufälligen WEPkey:

#!/bin/sh
#@@CCK 2003-11-19 14:16
#
# Output of "$1" (default: 8) random bytes

DEFAULTNUM=8

NUM="$1"
[ -z "$NUM" ] && NUM="$DEFAULTNUM"
[ "$NUM" -lt 0 ] && NUM=$(( -1 * $NUM ))

dd if=/dev/urandom bs=1 count=$NUM 2> /dev/null 
| hexdump 
| cut -c 9- 
| sed -e :a -e '/$/N; s/n/ /; ta' 
| sed -e '/^ *$/ d' -e 's/ //g' 
| cut -c -$(( 2 * $NUM ))

Hat man mit

# randombytes 13 > /etc/WEPkey
# chmod 600 /etc/WEPkey

einen WEPkey vorliegen und diesen auch bereits im Accesspoint eingetragen, so wird die Karte mit dem Kommando

# iwconfig ath0 essid WLAN enc `cat /etc/WEPkey`
# ifconfig ath0 192.168.0.17 up

konfiguriert - wobei die Kennung "WLAN" und die IP-Adresse hier nur als Beispiel dienen.

Wer WPA einsetzen will muss noch etwas Aufwand betreiben. Eine recht gute Beschreibung findet man auf www.fehu.org. Die wichtigsten Schritte wollen wir hier kurz darstellen:
Wichtige Voraussetzung: Die libssl-dev muss installiert sein, außerdem muss man wpa_supplicant herunterladen und entpacken: 

cd /usr/scr
$ wget http://hostap.epitest.fi/releases/wpa_supplicant-0.3.8.tar.gz
$ tar xvzf wpa_supplicant-0.3.8.tar.gz
$ cd wpa_supplicant-0.3.8
Nun muß man in diesem Verzeichnis eine .config anlegen mit folgendem Inhalt: 
CONFIG_DRIVER_MADWIFI=y
CFLAGS += -I/usr/src/madwifi
CONFIG_DRIVER_WEXT=y
CONFIG_IEEE8021X_EAPOL=y
CONFIG_EAP_MD5=y
CONFIG_EAP_MSCHAPV2=y
CONFIG_EAP_TLS=y
CONFIG_EAP_PEAP=y
CONFIG_EAP_TTLS=y
CONFIG_EAP_GTC=y
CONFIG_EAP_OTP=y
CONFIG_EAP_LEAP=y
CONFIG_PKCS12=y
CONFIG_CTRL_IFACE=y
Damit kann man den Treiber kompilieren: 
make && make install
Nun sollte man die wpa_supplicant.conf editieren und nach /etc kopieren. Inhalt der wpa_supplicant.conf: 
ctrl_interface=/var/run/wpa_supplicant
eapol_version=1
network={
ssid="fehu.org"
scan_ssid=1
priority=5
proto=WPA
key_mgmt=WPA-PSK
pairwise=CCMP TKIP
group=CCMP TKIP WEP104 WEP40
    #psk="mysecretpassword"
    psk=283fe364eb0ce1352c1831dee1ed08481eda8ba62583d6c6589a386cbd6957af
    }
Der Key hinter psk wird mit dem Tool wpa_passphrase erzeugt. Startet man nun wpa_supplicant verbindet sich die Karte nach ein paar Sekunden mit dem Access-Point: 
$ wpa_supplicant -w -i ath0 -c /etc/wpa-fehu.conf -D madwifi
